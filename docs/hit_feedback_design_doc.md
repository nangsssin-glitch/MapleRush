# 타격감(Hit Feedback) 기획서

> **문서 목적**: 메이플스토리 월드(MSW) 기반 프로젝트 "메이플 러시(메커톤)"에서 **플레이어가 적을 타격했을 때의 타격감**을 어떻게 표현·적용할지 정의한다. 핵심은 **이나리(Inari) / 카타나 제로(Katana Zero)** 류의 "한 방 한 방이 묵직하게 꽂히는" 느낌이며, 그 중심 연출은 **타격 시 화면이 살짝 흔들리는 카메라 셰이크**다.
> **연계 문서**: `stage_design_doc_ver3.md` (적 행동·전투 구조), (예정) `player_design_doc.md` (플레이어 스탯·공격·패링)
> **대상 독자**: 본 기획서를 바탕으로 구현하는 개발자
> **범위 주의**: 본 문서는 **무엇을 어떤 강도로 보여줄지(기획)**만 정의한다. 구체적인 구현 방법(API·코드·구조)은 개발자 재량에 맡긴다.
> **최종 수정일**: 2026-06 ver3 (싱글플레이 기준 확정 / 넉백 제외·피격 경직(0.1초)으로 변경 / 데미지 숫자 제외(별도 기획서) / 셰이크·히트스톱 값 컴포넌트 노출 요구 추가)

## 한눈에 보기 (5문답)

> 문서 전체를 읽지 않아도 핵심을 파악할 수 있도록 5문답으로 요약한다. 상세는 각 섹션 참조.

**Q1. 무엇을 만드나요?**
플레이어가 **적을 때렸을 때** "묵직하게 꽂히는" 손맛(타격감)을 만든다. 레퍼런스는 *이나리* · *카타나 제로*. → §1

**Q2. 어떤 요소로 표현하나요?**
4가지가 **동시에** 발동한다 — ① **카메라 셰이크**(화면 살짝 흔들림) ② **히트 스톱**(화면 전체 순간 정지) ③ **피격 경직**(맞은 적만 잠깐 멈춤) ④ **타격음**. (+선택: 충돌 이펙트) → §2

**Q3. 얼마나 지속되나요?**
타격 순간부터 **총 0.1초** 안에 시작·복귀한다. 화면 정지 0.05초, 셰이크·적 경직 0.1초. 길게 끌면 답답하므로 짧고 빠르게 복귀. → §3·§4

**Q4. 강도는 어떻게 정하나요?**
**모든 타격에 동일한 강도**(크리티컬·처치 등급 구분 없음). 셰이크 강도·히트스톱 시간은 **기획자가 코드 없이 컴포넌트에서 직접 조정**하며 실제 플레이로 튜닝. → §3

**Q5. 무엇은 하지 않나요?**
**넉백 없음**(경직만), **데미지 숫자·피격 플래시·처치 슬로우·카메라 줌 없음**, **플레이어 피격 연출 없음**(한 대 = 즉사), **싱글플레이** 기준. → §2·§5

## 1. 타격감 설계 철학

### 1.1 레퍼런스 분석 — 무엇이 "타격감"을 만드는가

| 게임 | 핵심 연출 | 우리가 가져올 것 |
|---|---|---|
| **카타나 제로** | 타격 순간 **짧은 화면 정지(히트 스톱)**. "끊었다"는 손맛이 강함 | 히트 스톱 |
| **이나리** | 타격마다 **즉각적이고 짧은 카메라 셰이크** + 경쾌한 타격음. 흔들림이 과하지 않고 빠르게 복귀 | 카메라 셰이크(짧고 빠른 복귀) + 타격음 |

> **핵심 통찰**: 사용자가 원한 "화면이 살짝 흔들리는" 카메라 셰이크는 타격감의 **얼굴**이지만, 그 두 레퍼런스가 진짜 "묵직하다"고 느껴지는 결정적 요인은 **히트 스톱(타격 순간 아주 짧게 시간이 멈췄다 풀리는 것)**이다. 카메라 셰이크 단독으로는 "흔들리기만 하고 가벼운" 느낌이 나기 쉽다. **따라서 본 기획은 카메라 셰이크를 중심에 두되, 히트 스톱을 동급의 필수 요소로 함께 설계한다.**

### 1.2 설계 원칙

1. **레이어드(중첩) 연출**: 하나의 타격에 여러 요소(셰이크·히트스톱·이펙트·사운드)가 **동시에** 겹쳐 발동한다. 단일 요소는 약하고, 합쳐질 때 "타격감"이 된다.
2. **짧고 빠른 복귀**: 타격감 전체는 **총 0.1초** 안에 끝나고 원상복귀한다. 길면 "조작이 답답하다"는 역효과가 난다. (이나리/카타나 제로 공통 특징)
3. **단일 강도(등급 구분 없음)**: 일반 타격·크리티컬·처치를 구분하지 않고, **모든 타격에 동일한 타격감**을 적용한다. (단순·일관성 우선)
4. **과잉 방지(누적 차단)**: 같은 순간에 여러 적을 동시에 때려도 셰이크가 합산돼 **화면이 멀미나게 흔들리면 안 된다.** 흔들림에 상한을 둔다.

## 2. 타격감 구성 요소

플레이어의 공격이 적에게 적중(타격 판정 성립)하면 아래 요소가 **동시에** 발동한다. "필수"는 MVP에 반드시, "권장"은 폴리싱 단계에 적용한다.

| 요소 | 역할 | 우선순위 |
|---|---|:---:|
| **카메라 셰이크** | 화면이 살짝 흔들려 충격을 전달 (핵심 요청) | **필수** |
| **히트 스톱** | 타격 순간 아주 짧게 **화면 전체** 시간을 멈췄다 풀어 "꽂히는" 손맛 | **필수** |
| **피격 경직** | 맞은 **적이 그 자리에서 0.1초 멈춤** → 타격이 적중했다는 반응 (넉백 없음) | **필수** |
| **타격음(SFX)** | 청각적 임팩트 | **필수** |
| **타격 이펙트(VFX)** | 충돌 지점에 스파크/임팩트 표시 | 권장 |

> **⚠ "히트 스톱"과 "피격 경직"은 다른 것이다.**
> - **히트 스톱** = **화면 전체**가 아주 짧게(0.05초) 멈췄다 풀리는 연출. 플레이어 손맛용.
> - **피격 경직** = **맞은 그 적 하나만** 0.1초간 이동/행동을 멈추는 전투 반응. 넉백(밀려남)은 적용하지 않는다.
>
> **MVP 최소 구성** = 카메라 셰이크 + 히트 스톱 + 피격 경직 + 타격음.
>
> **범위 제외**: 피격 대상이 번쩍이는 **피격 플래시/히트 이펙트는 엔진 기본 기능으로 자동 처리**되므로 본 문서에서 별도로 설계하지 않는다. **처치 슬로우 모션, 카메라 줌**은 게임 템포를 해칠 수 있어 본 버전에서는 **적용하지 않는다.** **데미지 숫자 표시**는 본 문서 범위 외이며 **추후 별도 기획서**로 작성한다. **넉백은 적용하지 않는다**(경직으로 대체). 본 게임은 **싱글플레이** 기준이다.

## 3. 타격감 파라미터

모든 타격에 동일하게 적용하는 **단일 값**이다. 타격감 전체는 **타격 순간부터 0.1초 안**에 발동·복귀한다.

> ⚠ **튜닝 전제**: 카메라 셰이크 강도는 게임마다 체감이 크게 다르므로, 아래 값은 **"시작값"**이며 **실제 플레이로 반드시 캘리브레이션**해야 한다. "거슬리지 않는 최소한의 흔들림"을 기준으로 잡는다.
>
> ⭐ **요구사항(개발자 전달)**: **카메라 셰이크 강도**와 **히트 스톱 정지시간**은 기획자가 **코드 수정 없이 컴포넌트 속성(인스펙터)에서 직접 조정**할 수 있도록 노출할 것. (캘리브레이션을 기획자가 반복해야 하므로)

| 요소 | 파라미터 | 시작값 | 비고 |
|---|---|:---:|---|
| 카메라 셰이크 | 강도(intensity) | 0.15 | ⭐컴포넌트 노출. 작게 시작해 체감되는 최소치로 조정 |
| 카메라 셰이크 | 지속시간 | 0.10초 | 총 발동시간과 동일 — 즉시 복귀 |
| 히트 스톱 | 정지시간(freeze) | 0.05초 | ⭐컴포넌트 노출. 0.1초 구간 내, 너무 길면 답답함 |
| 피격 경직 | 적 정지시간 | 0.10초 | 맞은 적만 이동/행동 정지. 넉백 없음 |
| 타격 이펙트 | 종류 | 충돌 스파크 1종 | (권장) 비주얼 에셋은 추후 지정 |
| 타격음 | 재생 | 타격 즉시 1회 | (필수) 사운드 에셋 추후 지정 |

## 4. 단일 타격 발동 시퀀스

타격 판정이 성립한 순간(t=0)을 기준으로, 모든 연출이 **0.1초 안**에 시작·복귀한다.

```
t=0.00   [타격 적중]
         ├─ 히트 스톱 시작 (화면 전체가 "탁" 멈춤)
         ├─ 카메라 셰이크 시작 (살짝 흔들림)
         ├─ 피격 경직 시작 (맞은 적이 그 자리에서 정지)
         ├─ 타격 이펙트 표시 (충돌 지점 스파크)
         └─ 타격음 재생

t=0.05   [히트 스톱 해제] 화면 전체 시간 정상 복귀 → 손맛

t=0.10   [카메라 셰이크 종료] 화면 완전 복귀
         [피격 경직 해제] 적 행동 재개 → 타격감 종료
```

> 모든 요소는 동시에 시작한다. 히트 스톱(0.05초)이 먼저 풀려 손맛을 주고, 카메라 셰이크(0.10초)와 피격 경직(0.10초)이 마지막에 종료되어 **총 0.1초**에 모든 연출이 끝난다.

## 5. 예외·주의 사항

| 상황 | 처리 방침 |
|---|---|
| **같은 순간 다중 타격(광역 공격 등)** | 셰이크가 합산돼 과도하게 흔들리지 않도록 **한 번의 흔들림으로 제한**한다. 히트 스톱도 1회만 적용. (단, 피격 경직은 맞은 적 각각에 개별 적용) |
| **빠른 연타** | 히트 스톱이 연달아 걸리면 조작이 답답해질 수 있으므로, 연타 시에는 정지시간을 줄이거나 최소 간격을 둔다. |
| **타격감 발동 범위** | 본 문서는 **플레이어가 적을 타격할 때**만 다룬다. 플레이어는 **한 대 맞으면 즉사**하므로, 플레이어가 피격당할 때의 타격감 연출은 **적용하지 않는다**. |
| **싱글플레이 전제** | 본 게임은 싱글플레이다. 따라서 "누구 화면을 흔드나" 같은 멀티플레이 동기화 고려는 필요 없다. |

## 6. 수용 기준(QA 체크리스트)

실제 플레이에서 아래가 만족되면 합격으로 본다.

- [ ] 적을 타격하면 카메라가 **살짝, 짧게(0.1초 이내)** 흔들리고 즉시 복귀한다
- [ ] 타격 순간 화면이 **아주 짧게 멈췄다(히트 스톱)** 풀린다 — "탁" 꽂히는 손맛이 느껴진다
- [ ] 맞은 적이 **그 자리에서 0.1초 멈췄다(경직)** 다시 움직인다 — 밀려나지(넉백) 않는다
- [ ] 타격음이 타격과 동시에 재생된다
- [ ] 타격감 발동 후 게임이 **정상 속도로 복귀**한다 (느려지거나 멈춘 상태로 남지 않는다) ★최우선
- [ ] 광역 공격이나 빠른 연타로 여러 번 맞춰도 **화면이 멀미나게 누적 흔들리지 않는다**
- [ ] 흔들림 강도가 "거슬리지 않으면서도 체감되는" 수준으로 튜닝되어 있다
- [ ] 기획자가 셰이크 강도·히트 스톱 시간을 **컴포넌트에서 코드 없이 조정**할 수 있다

## 7. 향후 확장

- **데미지 숫자 표시**: 본 문서 범위 외. **별도 기획서**로 작성 예정.
- **접근성 옵션**: "화면 흔들림 줄이기/끄기" 설정 — 멀미 민감 사용자 대응.
- **패링 성공 연출**: `stage_design_doc`의 패링(노랑 공격) 성공 시, 일반 타격보다 강한 별도 연출 검토.
- **보스 전용 연출**: 보스 처치 등 특별한 순간에 한해 더 강한 "피니시" 연출 검토 (`MR-B`/`MR-K` 보스 연계).
