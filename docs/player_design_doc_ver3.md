## 1. 플레이어 스탯

### 1.1 스탯 목록 및 기본값

| 스탯 이름 | 설명 | 기본값 | 비고 |
|---|---|---|---|
| `공격력` | 모든 공격(평타, 대시, 능력)에 공통 적용되는 수치 | **10** | 백분율 기반 데미지 계산의 기준값 |
| `목숨` | 스테이지 재시도 가능 횟수 (≠ HP, 1피격 즉사) | **5** | 0이 된 상태에서 추가 피격 시 게임 오버. **각 스테이지 진입 시마다 최대치로 갱신됨** (§1.4 참조) |
| `최대 대시 횟수` | 연속으로 사용할 수 있는 대시의 최대 횟수 | 2 | 게임 내 획득/강화로 증가 가능 |
| `대시 회복 시간` | 소모된 대시 횟수가 1회 회복되기까지 걸리는 시간 | 2.5초 | 게임 내 획득/강화로 단축 가능 |
| `공격 범위` | 평타 및 능력의 공격 판정 범위(스탯) | — | **미도입 확정** — 공격별 고정값을 하드코딩으로 사용 (§1.3 참조) |
| `최대 아이템 보관 수` | 일회성 아이템을 동시에 보유할 수 있는 최대 개수 | **3** | 초과 획득 시 버리거나 교체 처리 필요 |
| `플레이어 크기` | 캐릭터 크기 (2등신). 게임 내 **모든 범위 계산의 기준 base = 가로 1** | **가로 1 × 세로 2** (world unit) | 평타·감지·은신 등 "플레이어 크기 ×N배"는 base 1(가로) 기준 → **플레이어 중심 반지름 N world unit 원형** (§2.2·§3.2.1 등) |

> **범위 반지름 정의(플레이어 중심 원형, world unit)** — 평타 판정 **2** / 슬로우 트리거 **1** / 근거리 적 감지 **3** / 근거리 적 공격 **1** / 원거리 적 감지 **5** / 투사체 히트박스 **0.2(=1/5)** / 연막탄 은신 **3**.

### 1.2 스탯 증가 방법

- **게임 판 내 임시 증가**: 일반/보스 스테이지 클리어 보상, 상점 스테이지 구입
- **영구 증가**: 게임 전체 클리어 시 획득하는 점수로 메타 진행 트리에서 영구 강화 가능

### 1.3 예외 처리

> - `목숨`은 HP가 아니므로 피격 즉시 사망 처리하고 목숨 1 감소, 현재 스테이지 재시작
> - `목숨`이 0인 상태에서 추가 피격 시 게임 오버 처리 (타이틀 또는 결과 화면으로 이동)
> - `공격 범위` 스탯은 **미도입 확정**; 평타·능력별 공격 판정 범위는 각각 하드코딩된 고정값을 사용 (평타 판정 범위: 플레이어 크기의 **2배 원형** — §2.2 참조)
> - `최대 아이템 보관 수(3칸)` 초과 시 아이템 획득 불가; 버리기 또는 교체 선택 UI 표시

---

### 1.4 목숨 갱신 규칙 (재시도 한도 방식)

`목숨`은 누적 HP가 아니라 **스테이지 단위 재시도 한도**로 동작한다. 1피격 즉사 규칙과 함께 게임 난이도를 조절하는 핵심 장치이다.

| 시점 | 처리 |
|---|---|
| 스테이지 진입(클리어 후 다음 스테이지 진입 포함) | 목숨을 **현재 최대치로 갱신(복구)** |
| 스테이지 내 플레이어 피격(사망) | 목숨 1 감소 후 **현재 스테이지 재시작** |
| 한 스테이지 내에서 목숨 0 도달 후 추가 피격 | **게임 오버** |
| 상점에서 `최대 목숨 추가` 구입 | **최대 목숨 +1** (상점도 진입 시 최대치로 갱신되어 있으므로 현재 목숨도 즉시 +1 적용) (`stage_design_doc.md` §2.4 참조) |

> - 목숨은 스테이지를 넘어 누적·이월되지 않으며, 각 스테이지 진입 시 항상 최대치에서 시작한다.
> - 따라서 게임 오버는 **하나의 스테이지 안에서 최대 목숨만큼 사망**했을 때에만 발생한다.
> - 영구 강화 또는 상점 구입으로 최대 목숨이 변경된 경우, 이후 모든 스테이지 진입 시 변경된 최대치로 갱신된다 (`stage_design_doc.md` §4.5 참조).

## 2. 기본 공격 (평타)

### 2.1 조작 방법

| 입력 | 동작 |
|---|---|
| **마우스 좌클릭** | 마우스 커서 방향으로 근접 공격 판정 발생 |

### 2.2 공격 특성

- **캐릭터**: 만지(Manji) — 근접 즉시 판정 방식
- **공격 방향**: 플레이어 → 마우스 커서 방향 (360° 자유 방향)
- **공격 방식**: 근접 즉시 판정 (투사체 없음, 클릭 시 해당 방향 판정 범위 내 즉시 판정)
- **판정 범위**: **플레이어 중심 기준 반지름 2 world unit 원형** (= "플레이어 크기(가로 1)의 2배"; 고정값, `공격 범위` 스탯 미도입으로 하드코딩)
- **데미지**: `공격력 × 100%` (기본값 기준: 10 × 100% = **10**)
- **쿨타임**: **0.5초** (이전 평타 사용 후 0.5초 이내 추가 입력 무시)

### 2.3 패링 연계

- 평타 판정이 적의 **노란색 공격**과 맞닿으면 패링 발동 → [5장 패링 시스템](#5-패링-시스템) 참조

### 2.4 예외 처리

> - 대시 이동 중에는 평타 사용 불가; 대시 이동 완료 후 즉시 사용 가능
> - 슬로우 모션 중 평타 사용 시 슬로우 모션이 강제 해제되지 않음
> - 평타로는 패링이 가능하나, 대시 공격으로는 패링 불가
> - 평타의 근접 판정 범위는 **플레이어 크기의 2배 원형(고정값)** — 마우스 커서 방향을 중심으로 판정 (`공격 범위` 스탯 미도입)
> - 쿨타임(0.5초) 중 입력은 무시 처리; 쿨타임 종료 직후 입력은 즉시 발동
> - 슬로우 모션 중 평타 쿨타임은 **실제 시간 기준 0.5초**로 고정 (게임 내 시간 배율 영향 없음)

## 3. 대시 & 슬로우 모션

### 3.1 대시

#### 3.1.1 조작 방법

| 입력 | 동작 |
|---|---|
| **마우스 우클릭 유지** | 대시 방향 조준 (360° 자유 조정 가능) |
| **마우스 우클릭 해제** | 대시 실행 |

#### 3.1.2 대시 기본 사양

| 항목 | 값 | 비고 |
|---|---|---|
| 최대 사용 횟수 | 2회 | 스탯 강화로 증가 가능 |
| 회복 시간 | 2.5초 / 1회 | 스탯 강화로 단축 가능 |
| 이동 방향 | 360° | 우클릭 유지 중 자유 조정 |
| 이동 거리 | 플레이어 캐릭터 5체 길이 | 고정값 |
| 이동 소요 시간 | 0.1초 | 고정값 |
| **무적(i-frame)** | **이동 시간(0.1초) 동안 무적** | 피격·즉사 판정 무시; **강화 대시도 동일** |
| 대시 데미지 | `공격력 × 50%` (기본값 기준: **5**) | 이동 경로 위의 적에게 적용 |

#### 3.1.3 예외 처리

> - 대시 이동 중에는 평타 사용 불가; 이동 완료 후 즉시 가능
> - 대시 공격은 패링을 발동시키지 않음
> - 대시 횟수가 0인 상태에서 대시 입력 시 아무런 동작 없음 (UI 피드백 필요)
> - **대시 이동 시간(0.1초) 동안 플레이어는 무적(i-frame)** — 이 시간에는 적 공격/투사체와 겹쳐도 **피격·즉사되지 않으며**, 무적은 대시 이동 종료 **즉시 해제**된다 (실제 시간 0.1초 기준, 슬로우 모션 무관). **강화 대시도 동일하게 무적.** (1피격 즉사 규칙의 예외 — §1.3 참조)
> - 슬로우 모션은 **대시 발동 순간 트리거 범위(반지름 1) 내에 적 공격이 있을 때**만 발동하며, 대시 무적과는 **별개 판정**이다 (무적 = 대시 자체의 효과 / 슬로우 = 회피 성공 보상)
> - 대시 회복 타이머는 대시 사용 직후부터 카운트 시작; 대시 횟수가 최대치일 때는 타이머 정지

### 3.2 슬로우 모션

#### 3.2.1 발동 조건

적의 공격이 **플레이어 중심 기준 반지름 1 world unit 원형 감지 범위(= 플레이어 크기 1배)** 내로 진입한 순간 대시를 사용하면 슬로우 모션이 자동 발동된다.

| 구분 | 조건 상세 |
|---|---|
| 근거리 적 공격 | 적의 근거리 공격 판정이 플레이어 기준 원형 감지 범위 내에 진입 시 + 대시 사용 |
| 원거리 투사체 | 날아오는 투사체가 플레이어 기준 원형 감지 범위 내에 진입 시 + 대시 사용 |

> **감지 범위 기준**: 플레이어 캐릭터 크기와 동일한 반지름의 원형 (히트박스와 별도인 "슬로우 모션 트리거 범위")

- 참고 레퍼런스: 퍼니싱 그레이 레이븐의 초산 회피 슬로우 모션

#### 3.2.2 슬로우 모션 효과

| 항목 | 내용 |
|---|---|
| 지속 시간 | 1초 (실제 시간 기준) |
| 슬로우 배속 | 0.25배속 |
| 슬로우 **적용** 대상 | 적 이동/공격, 투사체, 게임 이펙트 등 플레이어가 조작하지 않는 모든 요소 |
| 슬로우 **미적용** 대상 | 플레이어가 조작하는 모든 것 + 카메라 (아래 표 참조) |
| 부가 효과 | 대시가 **강화 대시**로 1회 교체됨 (아래 참조) |

**슬로우 모션 중 실제 시간 기준으로 동작하는 요소 (슬로우 미적용)**

| 요소 | 비고 |
|---|---|
| 플레이어 이동 속도 | 정상 속도 유지 |
| 플레이어 점프 | 정상 속도 유지 |
| 평타 공격 판정 발생 속도 | 정상 속도 유지 |
| 평타 쿨타임 | 실제 시간 기준 0.5초 고정 |
| 평타 패링 판정 처리 | 충돌 감지는 실제 시간 기준 프레임으로 처리 (적 히트박스는 슬로우 적용되나 패링 판정 자체는 실제 시간 기준) |
| 대시 이동 속도 | 정상 속도 유지 |
| 대시 이동 소요 시간 | 실제 시간 기준 0.1초 고정 |
| 대시 회복 시간 | 실제 시간 기준 2.5초 고정 |
| 강화 대시 이동 및 피해 판정 | 강화 대시 사용 즉시 슬로우 해제되나, 해제 전 발동된 강화 대시의 이동 거리·데미지 판정은 실제 시간 기준으로 처리 |
| 카메라 이동 속도 | 정상 속도 유지 |
| 능력 발동 속도 및 지속 시간 | 능력 발동·사용 횟수 소모·지속 시간(예: 임시 무적 1초)은 실제 시간 기준으로 처리 |
| HUD / UI 갱신 속도 | 대시 횟수 게이지, 능력 아이콘, 목숨 표시 등 모든 UI 업데이트는 실제 시간 기준 (MSW: `_UtilLogic:SetClientTimeScale` 영향 차단 필요) |
| 플레이어 액션 사운드 재생 속도 | 평타 타격음, 대시 이동음·타격음, 점프음 등 플레이어 조작 관련 사운드는 정상 배속 재생 (슬로우 배속 영향 없음) |

#### 3.2.3 강화 대시

슬로우 모션 발동 시, 다음 1회 대시에 한해 일반 대시가 강화 대시로 교체된다.

| 항목 | 일반 대시 | 강화 대시 |
|---|---|---|
| 데미지 | `공격력 × 50%` | `공격력 × 150%` (기본값 기준: **15**) |
| 대시 횟수 소모 | O (1회 소모) | X (소모 없음) |
| 사용 가능 조건 | 대시 횟수 1회 이상 | 슬로우 모션 지속 중 |

#### 3.2.4 슬로우 모션 강제 해제 조건

| 조건 | 강제 해제 여부 |
|---|---|
| 슬로우 모션 중 **강화 대시 사용** | ✅ 즉시 해제 |
| 슬로우 모션 중 **평타 사용** | ❌ 해제 안 됨 |
| 지속 시간 1초 경과 | ✅ 자동 해제 |

#### 3.2.5 예외 처리

> - 슬로우 모션 트리거 범위(원형)는 적의 실제 히트박스와 별도로 구현; 해당 범위에 진입해도 실제 피격은 플레이어 히트박스 기준으로 판정
> - 슬로우 모션 발동 중 재차 슬로우 모션 조건을 충족해도 중첩되지 않고 지속 시간이 초기화(갱신)됨
> - 강화 대시를 사용하지 않고 슬로우 모션 시간이 종료되면 강화 대시는 소멸 (일반 대시로 복귀)
> - 강화 대시는 대시 횟수를 소모하지 않으므로, 대시 횟수가 0인 상태에서도 사용 가능
> - 슬로우 모션 트리거 범위 내에 여러 공격이 동시에 진입한 경우, 첫 번째 진입 공격을 기준으로 1회 발동
> - 슬로우 모션의 1초 지속 시간은 게임 내 시간이 아닌 **실제 시간(Wall-clock time)** 기준으로 측정
> - 플레이어가 조작하는 모든 요소(이동, 점프, 평타, 대시, 카메라, 능력, HUD/UI, 플레이어 액션 사운드)는 게임 내 시간 스케일과 독립적으로 실제 시간 기준으로 동작하도록 구현
> - 슬로우 모션 중 플레이어 조작 요소와 적 요소의 시간 스케일이 달라지므로, 충돌 판정(히트박스 겹침 감지)은 실제 시간 기준 프레임으로 처리해야 정확도 유지 가능
>
> **[MSW 엔진 구현 주의사항]** *(ver3 수정: 실제 API 명칭·서버 권위·슬로우 정리 로직 반영)*
> - **슬로우 모션 API 정정(B-1)**: 기존 문서의 `_CameraService.TimeScale`은 **실제 MSW에 존재하지 않는 속성**이다. Unity의 `Time.timeScale`에 해당하는 올바른 API는 **`_UtilLogic:SetClientTimeScale(float)`** 이다 (클라이언트 전용, 예: `_UtilLogic:SetClientTimeScale(0.25)`; 기본값 1.0, 0~100 범위).
> - `SetClientTimeScale`은 **`OnUpdate(delta)`로 전달되는 deltaTime 값만 변경**하며, 클라이언트의 실제 실행 속도나 서버 시뮬레이션에는 영향을 주지 않는다. 또한 **룸(맵) 이동 시 자동으로 1.0으로 리셋**된다.
> - ⚠️ **스프라이트 애니메이션 자동 감속은 보장되지 않음(C-2)**: `SetClientTimeScale`이 스프라이트 애니메이션 재생 속도까지 함께 늦추는지는 **확정되지 않았다**. 따라서 적 애니메이션 감속은 자동 동작에 의존하지 말고 **각 애니메이션 컴포넌트의 `PlayRate`를 직접 0.25로 설정**하는 방식을 기본으로 한다.
> - **적·투사체 감속은 서버 권위 기준으로 처리(C-3)**: 적과 적 투사체의 위치는 서버에서 시뮬레이션·동기화되므로 클라이언트 `SetClientTimeScale`만으로는 실제 이동이 느려지지 않는다. **현재 진행 구조(1인 1런: 전투 인스턴스에 플레이어 1명) 기준으로, 슬로우 발동 시 서버에 요청(`@ExecSpace("Server")`)하여 현재 맵의 모든 적·투사체의 `MovementComponent.InputSpeed`/투사체 속도/애니메이션 `SpriteRendererComponent.PlayRate`를 0.25배로 직접 조정**한다(프로퍼티 직접 제어 방식 확정). 플레이어 조작 요소는 이 조정에서 제외한다. ✅ **검증완료(V1 실측)**: 서버에서 `InputSpeed`를 0.25배로 설정하니 이동 속도가 정확히 0.25배(0.62/s → 0.16/s)로 감속되고 1.0 복귀 시 정상 복구됨을 확인. `PlayRate`도 런타임 설정 가능(오류 없음).
>   - 🔸 **코옵(스테이지 공유) 미정 주석**: 추후 여러 플레이어가 같은 전투 스테이지를 공유하는 코옵이 도입되면, 위의 "서버 전역 적 감속"은 **다른 플레이어에게도 적이 느려지는 잘못된 동작**이 된다. 그 경우 슬로우는 발동 플레이어의 **로컬 시각 전용**으로 바꾸고 서버 위치와의 보간·재동기화를 별도 구현해야 하며 난이도가 크게 상승한다. **코옵 작업 착수 전 반드시 재설계할 것.**
> - **플레이어 조작 요소는 Wall-clock 독립 타이머로 처리**: 이동·점프·평타·대시·카메라·능력·HUD/UI·플레이어 액션 사운드는 scaled delta에 의존하지 말고 실제 시간 기반 타이머로 정상 배속을 유지한다.
>   - ✅ **검증 완료(C-1, 실측)**: `_UtilLogic.ElapsedSeconds`와 `ServerElapsedSeconds`는 `SetClientTimeScale(0.25)`의 영향을 **받지 않고 실제 시간 그대로 흐른다**(8초 실측: real=8.01 ↔ ElapsedSeconds=8.02 ↔ ServerElapsed=8.02). **느려지는 것은 `OnUpdate(delta)`의 delta 값뿐**(정확히 0.25배). 따라서 → **플레이어 wall-clock 타이머는 `ElapsedSeconds` 스냅샷 차분으로 구현**한다(예: `if _UtilLogic.ElapsedSeconds - self.startTime >= 0.5 then ...`). ⛔ **`delta` 누산으로 구현 금지** — 그렇게 하면 슬로우 중 플레이어 타이머도 같이 0.25배로 느려진다. `DateTime.UtcNow` 대체는 불필요.
> - 슬로우 모션 지속 시간(1초), 평타 쿨타임(0.5초), 대시 이동 소요 시간(0.1초), 대시 회복 시간(2.5초), 능력 지속 시간 등 **실제 시간 기준이 필요한 모든 타이머는 위 독립 타이머**로 구현할 것.
> - **슬로우 종료·스테이지 전환 시 정리(A-2)**: 슬로우 지속(1초) 종료 즉시 `_UtilLogic:SetClientTimeScale(1.0)`로 복귀하고, 서버에 요청해 적·투사체 속도 배율과 `PlayRate`를 1.0으로 원복한다. **스테이지 진입·이탈·재시작 시에는** 슬로우가 진행 중이더라도 강제로 (1) 클라이언트 timeScale 1.0 복귀, (2) 모든 적 속도 배율·`PlayRate` 원복을 수행한다. (룸 이동 시 클라이언트 timeScale은 자동 리셋되지만, 서버측 적 속도 배율은 전환 직전 잔류 상태를 명시적으로 가드할 것. 재시작 시에는 적 재스폰으로 초기화됨 → `stage_design_doc.md` §2.3.5 단일 초기화 함수 참조.)
> - `SetClientTimeScale`을 장시간 임의로 변경하면 서버-클라이언트 위치 동기화 오차(데드 레코닝 오차)가 발생할 수 있으므로, 슬로우 지속(1초) 종료 즉시 1.0으로 복귀시킬 것.
> - **강화 대시 해제 시점**: 강화 대시 사용 즉시 `SetClientTimeScale(1.0)` 복귀 및 적 속도 원복; 강화 대시 이동(0.1초)이 완료되기 전 슬로우가 해제되어도 해당 대시의 이동 거리·데미지 판정은 Wall-clock 기준으로 끝까지 처리할 것

## 4. 능력 시스템

### 4.1 정의

강력한 효과를 발휘하는 비장의 수단. 스테이지마다 사용 횟수가 제한되며, 공격적 또는 유틸리티적으로 플레이어를 보조한다.

### 4.2 능력 슬롯 구조

- **슬롯 수**: 1개 (단일 슬롯)
- **교체 방식**: 새로운 능력을 획득하기로 선택하면 기존 능력이 즉시 소멸되고 새 능력으로 교체됨
- 교체는 되돌릴 수 없음 (교체 전 확인 UI 표시 권장)

### 4.3 조작 방법

| 입력 | 동작 |
|---|---|
| **마우스 휠 클릭 (가운데 버튼)** | 현재 장착된 능력 즉시 발동 |

### 4.4 사용 횟수 제한

| 스테이지 유형 | 스테이지당 최대 사용 횟수 |
|---|---|
| 일반 스테이지 | 1회 |
| 보스 스테이지 | 2회 |

### 4.5 능력 목록 (초안)

| 능력 이름 | 효과 | 강화 방향 (예시) |
|---|---|---|
| 임시 무적 | 1초 간 모든 피해 무효 | 지속 시간 증가 |
| 모든 투사체 무효화 | 화면 내 모든 투사체 즉시 제거 | 제거 범위 확장 또는 범위 내 추가 데미지 |
| (추가 예정) | - | - |

> 🔧 능력 목록은 추후 기획 확장 시 본 표에 항목 추가

### 4.6 능력 획득 방법

- 일반 스테이지 클리어 보상 (낮은 확률로 드롭)
- 상점 스테이지 구입
- 보스 스테이지 클리어 보상

### 4.7 예외 처리

> - 스테이지 진입 시 잔여 사용 횟수는 스테이지 기준 횟수로 초기화됨 (이전 스테이지 잔여 횟수 이월 불가)
> - 능력 미보유 상태에서 능력 입력 시 아무 동작 없음 (UI 피드백 필요)
> - 능력 사용 중(발동 이펙트 재생 중) 추가 능력 입력은 무시 처리
> - 보스 스테이지에서 2회 모두 사용 후 추가 입력 시 아무 동작 없음 (UI 피드백 필요)
> - 능력 보유 여부와 무관하게 조작 키는 항상 수신; 미보유/소진 시는 무시 처리로 통일
> - 능력 교체는 단방향으로 즉시 처리됨; 교체 후 되돌릴 수 없으므로 획득 선택 시 확인 UI(예: "현재 능력을 버리고 교체하시겠습니까?") 표시 권장
> - 능력 없는 상태에서 새 능력 획득 선택 시 바로 장착 (교체 확인 UI 불필요)
> - 스테이지 클리어 보상으로 능력이 제시될 때 "획득하지 않음(버림)" 선택지도 함께 표시

## 5. 패링 시스템

### 5.1 정의

기본 공격(평타)의 근접 판정이 적의 공격과 **물리적으로 맞닿는 순간** 자동으로 발동되는 시스템. 특정 공격을 무효화하거나 역이용할 수 있다.

### 5.2 적 공격 색상 구분

| 색상 | 의미 | 플레이어 대응 방법 |
|---|---|---|
| 🟡 **노란색** | 패링 가능한 공격 | 평타 판정을 맞닿게 하여 패링 발동 |
| 🔴 **빨간색** | 패링 불가 공격 | 회피(대시)만 가능 |

> **[코드 구현 — 갭② 해소]** 적 공격/투사체는 각각 **`parryable: boolean`** 플래그를 사전 지정한다 (true = 노란색·패링/반사 가능, false = 빨간색·패링 불가). 노랑/빨강 색상은 **추후 지정할 공격 이펙트의 시각 표현**이며, 패링/반사 판정 로직은 이 boolean 값으로 분기한다. (평타 원형 ↔ 적 공격 히트박스 오버랩은 §5.3, 오버랩 판정 동작은 V4 실측 확정)

### 5.3 패링 발동 조건

- 플레이어의 **평타 근접 판정 범위**와 적의 **공격 판정(근거리 히트박스 또는 투사체)**이 **서로 겹치는(맞닿는) 순간** 패링 발동
- 단, 대상 공격의 색상이 **노란색**이어야 함

### 5.4 패링 효과 상세

#### 5.4.1 근거리 노란색 공격 패링

| 항목 | 내용 |
|---|---|
| 발동 조건 | 평타 판정 범위와 적의 근거리 공격 히트박스가 겹치는 순간 |
| 효과 | 해당 공격 무효화 (플레이어 피격 없음) |
| 추가 데미지 | 없음 |
| 그로기 게이지 충전 (보스 한정) | 패링 발동 즉시 **일반 공격 명중 데미지(= `공격력 × 100%`)의 2배** 충전 (기본값 기준 10 × 2 = **20**); 공격력 강화 시 충전량도 비례 증가(의도된 설계) (`stage_design_doc.md` §1.4.2 참조) |

#### 5.4.2 원거리 노란색 공격 패링 (투사체 반사)

| 항목 | 내용 |
|---|---|
| 발동 조건 | 평타 판정 범위와 날아오는 투사체 히트박스가 겹치는 순간 |
| 효과 | 투사체를 발사한 적 방향으로 해당 투사체 반사 |
| 반사 데미지 | **`공격력 × 200%`** (기본값 기준: **20**) — **명중 대상 무관 통일** (원래 발사한 적/다른 적 모두 동일 적용) |
| 그로기 게이지 충전 (보스 한정) | 반사된 투사체가 보스에게 **실제로 명중**한 시점에 **일반 공격 명중 데미지(= `공격력 × 100%`)의 2배** 충전 (기본값 기준 **20**); 반사 직후 또는 명중 실패 시 충전 없음. 공격력 강화 시 충전량 비례 증가(의도) (`stage_design_doc.md` §1.4.2 참조) |

### 5.5 예외 처리

> - 패링은 **평타 근접 판정으로만** 발동 가능; 대시 이동 경로 데미지로는 패링 불가
> - 빨간색 공격에 평타가 맞닿아도 패링 미발동; 해당 공격이 그대로 플레이어에게 적용됨
> - 반사된 투사체는 플레이어에게 재피격되지 않음 (아군 피격 면제 처리)
> - 반사된 투사체는 원래 투사체와 동일한 히트박스 및 이동 속도 유지
> - 반사 후 투사체가 적에게 명중하지 못하고 화면 밖으로 벗어난 경우 자연 소멸 처리
> - 패링 성공/실패 여부를 플레이어가 즉시 인지할 수 있도록 시각/청각 피드백 필요 (이펙트, 사운드)
> - 투사체가 이미 화면 밖으로 나갔거나 소멸된 경우 패링 불가 처리
> - 그로기 게이지 충전은 **보스 스테이지에서만** 적용; 일반 적 상대 패링 시에는 그로기 게이지 충전 없음
> - 보스가 그로기 상태 중일 때 패링이 발동되더라도 그로기 게이지는 충전되지 않음 (그로기 해제 후부터 충전 재개)
> - **[MSW 구현 — 오버랩 판정 별도 구현(A-3)]** 평타 원형 판정과 적 공격 히트박스/투사체의 겹침(오버랩) 감지는 MSW의 자동 충돌(AttackComponent↔HitComponent) 파이프라인으로 직접 커버되지 않는다. 따라서 **평타 판정 원과 적 공격 히트박스/투사체의 거리·교차 검사를 별도 오버랩 로직으로 구현**한다. (원형 공격 범위 자체는 `AttackComponent` + Circle ColliderType으로 네이티브 표현 가능하나, "적 공격을 맞받아치는" 패링 판정은 커스텀.) 슬로우 모션 중에도 이 판정은 Wall-clock 기준 프레임으로 처리한다. ✅ **검증완료(V4 실측)**: 런타임에 두 대상의 위치를 읽어 원형 거리 오버랩 판정(`dist ≤ r1+r2`)이 정확히 true/false로 전환됨을 확인 — 물리 충돌 없이 수동 오버랩으로 패링/짱돌 판정 구현 가능. (남은 설계 의존성: 적 공격 히트박스의 위치·반경을 어떻게 노출할지)

## 6. 게임 재화 (메소)

### 6.1 정의

게임 진행 중 획득할 수 있는 기본 재화. 상점 스테이지에서 스펙업 및 보조 수단과 교환하는 데 사용된다.

### 6.2 재화 정보

| 항목 | 내용 |
|---|---|
| 재화 이름 | **메소 (Meso)** |
| 최댓값(캡) | **4,294,967,295** (unsigned int 최대값 기준) |
| 게임 판 간 이월 | 불가 (게임 오버 또는 클리어 시 소멸) |

### 6.3 획득 방법

- 적 처치 즉시 드롭되지 않으며, **스테이지 클리어 시 한꺼번에 지급**되는 방식
- 메소 지급 공식 (임시):
  - **일반 스테이지 클리어 시**: `일반 스테이지 클리어 수 × 10`
  - **보스 스테이지 클리어 시**: `보스 스테이지 클리어 수 × 150`

### 6.4 사용처

- 상점 스테이지에서 일회성 아이템, 스탯 강화, 능력, **최대 목숨 추가** 구입 → `stage_design_doc.md` §2.4 참조

### 6.5 예외 처리

> - 메소는 게임 판 내에서만 유효; 게임 오버 또는 클리어 시 소멸 (이월 불가)
> - 메소 최댓값(캡)은 **4,294,967,295** (unsigned int 최대값); 누적 결과가 이 값을 초과할 경우 **캡 값으로 클램프(clamp) 처리** (Lua는 더 큰 값을 저장할 수 있으나 본 게임에서는 이 값으로 상한 적용)
> - 상점 구입 시 잔액 부족이면 구입 불가 처리 (UI 피드백 필요)
> - 적 처치 즉시 드롭 없음; 스테이지 클리어 시 한꺼번에 자동 지급 (일반: `클리어 수×10`, 보스: `클리어 수×150`)

## 7. 일회성 아이템 사용

### 7.1 정의

게임 진행 중 보상·상점으로 획득하는 소모성 아이템. 사용 시 1회 소모되어 사라진다. 아이템별 효과·수치는 `stage_design_doc.md` §3.2.2에서 정의한다.

### 7.2 보관 슬롯 구조

| 항목 | 내용 |
|---|---|
| 최대 보관 수 | **3칸** (`최대 아이템 보관 수` 스탯, §1.1 참조) |
| 슬롯 번호 | **1번 · 2번 · 3번** 고정 (슬롯 자리 변경/정렬은 **미구현**) |
| 초과 획득 | 3칸이 가득 찬 상태에서 추가 획득 시 버리기/교체 UI 표시 (§1.3 참조) |

### 7.3 조작 방법

| 입력 | 동작 |
|---|---|
| **숫자키 1~3** | 해당 슬롯 아이템을 **활성화 상태**로 전환 (조준/사용 대기) |
| 활성화 상태에서 **마우스 좌클릭** | 해당 아이템을 **커서 방향으로 사용** 또는 즉시 적용 후 1회 소모 |
| 활성화 상태에서 **같은 숫자키 재입력** | **비활성화 상태**로 전환 (사용 취소) |

- 활성화는 한 번에 한 슬롯만 가능 (다른 숫자키 입력 시 활성 슬롯이 해당 슬롯으로 전환)
- 사용 방향이 필요 없는 즉시 적용형 아이템(예: 섬광탄)은 좌클릭 시 방향과 무관하게 즉시 발동

### 7.4 예외 처리

> - 평타(좌클릭)는 아이템이 **활성화된 상태에서는 아이템 사용 입력으로 처리**되며, 평타로 발동되지 않음 (활성화 중 좌클릭 우선순위 = 아이템 사용)
> - 아이템 사용 후 평타는 별도 제약 없이 **평타 쿨타임(0.5초)** 규칙을 그대로 따른다 (아이템 사용이 평타 쿨타임을 추가로 늘리거나 줄이지 않음; §2.2 참조). 즉 아이템 사용 → 슬롯 비활성화 → 이후 좌클릭은 0.5초 쿨타임 내라면 무시, 경과 후라면 정상 평타 발동
> - 빈 슬롯의 숫자키를 누르면 아무 동작 없음 (UI 피드백 권장)
> - 아이템 활성화 상태에서 대시(우클릭)·능력(휠클릭) 입력은 정상 동작하며, 활성화 상태는 유지됨
> - 아이템 사용으로 생성되는 효과(짱돌 투사체 등)는 플레이어가 발생시키므로 적의 패링 대상이 아님
> - 슬로우 모션 중 아이템 사용 입력 및 효과 발동도 실제 시간(Wall-clock) 기준으로 처리 (§3.2.2 슬로우 미적용 대상에 준함)
> - **아이템 효과 지속 시간은 실제 시간(Wall-clock) 기준**으로 측정 (연막탄 은신 5초, 섬광탄 일반 적 기절 3초, 섬광탄 보스 그로기 게이지 감소 정지 5초 등); 슬로우 모션(0.25배속)이 동시에 걸려 있어도 효과 지속 시간이 늘어나거나 줄지 않음 (`stage_design_doc.md` §3.2.2 참조)
> - **짱돌 투사체는 플레이어가 발생시키므로 슬로우 모션 영향을 받지 않음** (실제 시간 기준 정상 속도로 이동); 적 투사체 "사양"(속도·히트박스·소멸 규칙)만 재사용할 뿐, 슬로우 적용 대상이 아님
> - 대시 이동 중(0.1초)에는 좌클릭이 아이템 사용으로도 처리되지 않음 (평타와 동일하게 대시 이동 완료 후 가능; §3.1.3 참조)
> - 일회성 아이템은 **전투 스테이지(일반·보스)에서만 사용 가능**; 상점 스테이지 및 보상 UI가 열린 상태에서는 사용 입력 무시 (아이템 낭비 방지)
> - 아이템 사용 직후 슬롯은 비워지며 자동으로 비활성화 상태로 복귀
