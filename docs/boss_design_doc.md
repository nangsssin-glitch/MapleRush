## 1. 공통 설계 원칙

### 1.1 난이도 정책 — 3보스 대등

- 엘리쟈·스노우맨·자쿰은 **난이도 대등**(병렬 테마). HP·기본 시전속도(windup)·쿨타임 골격은 **동일**하게 두고, **패턴 구성·연출·히트박스 형태·이동성으로만 차별화**한다.
- (진행 순서에 따른 난이도 에스컬레이션은 **미적용** — 추후 층 확장 시 재검토.)

### 1.2 공통 수치 기준

> 용어는 `stage_design_doc_ver3.md` §1.4.2를 따른다. **"일반 공격 명중 데미지 = 공격력 × 100%"** 가 기준 단위이며, 패링 충전은 항상 **"일반 공격의 2배"** 로 표기한다(특정 숫자값으로 고정 표기하지 않음 — 공격력 강화 시 비례 증가).

| 항목 | 값 | 비고 |
|---|---|---|
| 보스 최대 HP | **800** (3보스 공통) | 튜닝값 |
| 그로기 게이지 최대치 | **보스 최대 HP의 1/5** | `stage_design` §1.4.2 자동 파생 |
| 일반 공격 명중 — HP 데미지 | 공격력 × 100% | 동시에 그로기를 **데미지량만큼** 충전 |
| 패링 그로기 충전(근접·원거리 공통) | **일반 공격의 2배** (= 공격력 × 200%) | 코드: `GroggyParryMult`. 숫자 고정 표기 금지 |
| → 패링만으로 그로기 발동까지 | 약 **8회**(기본 공격력 기준) | 일반 공격 병행 시 더 빠름. 공격력 강화 시 횟수 감소 |
| 그로기 게이지 자동 감소 | 초당 2% | 전투 중 일정, HP 무관 |
| 그로기 지속 | 3초 | 보스 이동·공격 완전 정지 |
| 투사체 속도(기본) | 10 | 적 투사체 사양 재사용(`stage_design` §1.3.2) |

> HP·시전속도·쿨타임은 플레이테스트(MR-E) 후 조정. 외부화 대상은 §4.2.

### 1.3 패링 색상 정책 (확정)

| 패턴 분류 | 색상 | parryable | 패링/반사 시 효과 |
|---|---|---|---|
| 근거리 패턴 | **노랑** | `true` | **HP는 깎이지 않음.** 패링 성공 시 그로기를 **일반 공격의 2배(= 공격력 × 200%)** 만큼 충전 + 공격 무효화 |
| 원거리(투사체) 패턴 | **노랑** | `true` | 반사 투사체가 보스에 **실제 명중** 시 `공격력 × 200%` 데미지 + 그로기 **일반 공격의 2배(= 공격력 × 200%)** 충전 |
| 광역(범위) 패턴 | **빨강** | `false` | 패링 불가 — **회피만 가능**. 예고 사각형 범위를 벗어나야 함 |

> 그로기 충전의 주력 수단은 근접·원거리 패링이며, 광역은 패링 루프를 끊는 회피 강제 패턴이다.

### 1.4 패링 메커니즘 및 타이밍 (코드 실태 — `PlayerCombat`/`CombatPrimitives`)

> 본 절은 **현재 구현된 동작**과 **의도(=이번 결정)** 의 차이를 명시한다. 패링 시스템은 이미 동작하므로, 보스 패턴은 이 메커니즘 위에서 설계한다.

**메커니즘 — 패링은 전용 키가 아니라 "좌클릭 평타"다.**

- 평타 시 서버(`RequestAttack`)가 플레이어 전방에 공격 박스(반폭 `PlayerAttackBoxForward/2` · 반높이 `PlayerAttackBoxHeight/2`)를 생성하고:
  - **근접(노랑) 패링**: 이 박스가 보스의 활성 근접 공격 박스(`parryable=true`)와 겹치면 → 그 공격을 **무효화(`canceled`)** + 해당 보스 `GroggyGauge`에 **공격력 × 200%** 충전. (`TryParry` → `gg:Charge(...)`)
  - **원거리(노랑) 패링(반사)**: 박스가 보스 투사체(`Parryable=true`)와 겹치면 → 투사체 **반사**(`comp:Reflect`). 반사체가 보스 명중 시 **공격력 × 200%** 데미지 + 그로기 충전.
- 즉 **위협을 향해 평타를 휘두르는 것이 곧 패링**이다(별도 패링 버튼 없음).

**타이밍 — 의도와 현재 코드의 차이 (★ 조정 필요)**

- 적 공격 박스는 windup 시작 시 등록되어 **`[등록 ~ 명중시각(hitAt) + 0.25s]`** 동안 살아 있고, 그 구간 내내 `parryable`이면 평타로 취소 가능.
- → **현재 코드: windup 도중 아무 때나 근접 패링 성공.** **"게이지 가득(모션 발생) 시점에만 패링"이라는 의도와 불일치.**
- → **원거리 패링은 의도와 자연히 일치**: 투사체는 resolve(발사 = 모션 발생) 후에야 존재 → 비행 중에만 반사 가능.
- **필요 조정(코드 — §5 TODO)**: 근접 패링 가능 구간을 모션 발생 순간(hitAt)으로 좁히는 **패링 윈도우** 도입.
  - **확정값: 선행 ε = 0.15s, 후행 = 0.05s** → 유효 구간 `hitAt − 0.15s ≤ now ≤ hitAt + 0.05s` (총 약 **0.2초**).
  - 목표 손맛 = **"약간 널널하되 예리"**: 모션 직전 0.15초부터 입력을 받아주되(널널), 모션 직후 0.05초에 거의 즉시 닫힘(예리). 미세 조정은 MR-E.
  - 구현: `TryParry`의 근접 분기에 `now`(ServerElapsedSeconds) vs `rec.hitAt` 시간 게이트 추가. (투사체 반사 분기는 변경 불필요.)

### 1.5 1피격 즉사와 예고(텔레그래프) 게이지

플레이어는 **한 대만 맞아도 사망**(1피격 즉사). 따라서 모든 패턴은 **반드시 회피 또는 패링 가능**해야 한다.

- **시전 속도(windup)** = 플레이어 반응/패링 입력 시간. 가장 빠른 패턴도 windup **0.6초 이상**.
- **구현 현황**: windup 타이머 + windup 동안 **공격 모션 재생**은 구현됨(`BossController`). 단 **시각 예고(범위 게이지)는 미구현** → 아래 게이지를 신규 구현(§5 TODO, 우선순위 높음).

**예고 게이지 규칙 (모든 보스 공통)** — 게이지가 차오르는 시간 = 해당 패턴의 windup, **가득 차는 순간 = resolve(패턴 발동 = 모션 발생 = 근접 패링 유효 시점)**.

| 패턴 분류 | 게이지 형태·위치 |
|---|---|
| **근거리 / 광역** | 공격이 발생할 **위치(바닥)** 에 **사각형 게이지가 아래 → 위로** 차오름. 가득 차면 그 **사각형 영역**에 패턴 발동 |
| **원거리(투사체)** | **보스 중심에서 바깥쪽(발사 방향)으로** 사각형 게이지가 차오름. 가득 차면 투사체 발사 |

- **예고 = 판정 영역 일치(사각형)**: 근거리·광역 패턴의 **실제 판정 히트박스는 예고 사각형과 동일한 사각 영역**으로 통일(공정성). 원형 판정 미사용. (코드: `RegisterEnemyAttackBox` / `DamageEnemiesInBox` / `BoxBoxOverlap` — 박스 판정 이미 지원.)
- **색상**: 노랑 패턴 = 노랑 게이지, 광역(빨강) = 빨강 게이지.
- **구현**: SpriteGUI 또는 월드 스프라이트 **Filled 모드**(근접·광역 = Vertical Bottom→Top, 원거리 = 보스 기준 바깥 방향).

### 1.6 패턴 선택 로직 (거리 밴드 + 우선순위)

기본 우선순위는 **광역 > 원거리 > 근접**이되, **플레이어와 보스의 가로 거리 d** 로 보정한다(요구사항 4-1 "거리에 따라 우선순위가 달라질 수 있음"). 각 밴드에서 **쿨타임이 끝난 최우선 패턴**을 선택하고, 그것이 쿨 중이면 다음 순위로 내려간다.

| 거리 밴드 (가로 거리 d) | 우선순위 |
|---|---|
| **CLOSE** — d ≤ 근접 발동 사거리 | 광역 > **근접** > 원거리 |
| **MID** — 근접 사거리 < d ≤ 광역 배치 사거리 | 광역 > 원거리 |
| **FAR** — d > 광역 배치 사거리 | 원거리 (광역 안 닿음 → 요구사항 4-2) |

- **★ 거리 밴드를 둔 이유(일관성 보정)**: 단순 "광역>원거리>근접 + 광역 안 닿으면 원거리"만 적용하면, **원거리는 사거리 제한이 없어** 광역이 쿨일 때 항상 원거리가 선택되고 **근접 패턴이 사실상 발동되지 않는다.** 그래서 **플레이어가 근접 사거리에 들어온 CLOSE 밴드에서만 근접 > 원거리**로 보정해 근접을 살린다(광역은 모든 밴드에서 최우선 유지).
- **광역 쿨타임은 의도적으로 길게**: 근접·원거리의 **2~3배**(§3: ~1.0s ↔ ~2.5s). 광역이 최우선이어도 자주 못 나와 "회피→딜" 루프 보장.
- **광역 배치 사거리(확정 6)**: 추적형 광역(암흑 강림·블리자드·용암 분출)은 **보스 중심 가로 6 world unit 이내**면 "닿음". 고정형 광역(그림자 쐐기 = 보스 양옆)은 보스 중심 ±5 도달. (미세 조정은 MR-E.)
- **엘리쟈**(근접 없음): CLOSE에서도 근접이 없어 광역>원거리만 적용.
- **현재 코드와 차이**: `BossController`는 단일 `phaseTimer` + 순수 랜덤(`RandomIntegerRange(1,3)`). 본 밴드·우선순위·**패턴별 개별 쿨타임**은 신규 구현(§5 TODO).

### 1.7 보스 이동/행동

- 보스는 플레이어를 **추격**한다(`move` 클립, MR-B). 추격 속도는 일반 적보다 느리게(현재 코드: 거리 1.8 초과 시 추격). **이 부분은 현재 코드 동작 유지로 충분.**
- 근접 패턴은 사거리 진입 시(§1.6 CLOSE), 원거리·광역은 §1.6 밴드 규칙에 따라 발동.
- 그로기 중에는 이동·공격·추격 전부 정지.
- **근접 발동 사거리 = 보스 개성·난이도 레버**: 보스마다 근접 발동 사거리가 달라 압박감(과 CLOSE 밴드 진입 거리)이 다르다.
  - 스노우맨 = 넓은 전방(가로 3.6, 느림) / 자쿰 = 긴 횡 사거리(가로 4, 좌우 택1) / 엘리쟈 = 근접 없음.
  - 사거리가 길수록 플레이어가 거리를 둬도 CLOSE에 들어가 근접에 노출 → 난이도↑. **패턴별 발동 사거리는 §3 박스 크기와 별도 파라미터**로 두고 외부화(§4.2).

## 2. 보스별 정의

3보스 모두 HP 800 / 그로기 최대치 = HP의 1/5 공통(§1.2). 차별점은 **패턴 구성·연출·히트박스 형태**.

### 2.1 오르비스 보스 — 엘리쟈(Elija) · 원거리+광역 회피형

| 항목 | 내용 |
|---|---|
| 테마 | 오르비스 — **어두운 세계관**에 어울리는 음산한 존재(어둠/그림자 컨셉) |
| 정체성 | **근접 패턴 없음**(개성). 원거리 1 + 광역 2의 **회피·견제형**. 거리를 두고 압박하다 광역으로 안전지대를 좁힌다 |
| 원거리 패턴 | **암흑 구체**(P-O1) — 유일한 노랑(패링 가능) 패턴, 3발 동시 |
| 광역 패턴 ① | **암흑 강림**(P-O2) — 빨강(회피 전용) |
| 광역 패턴 ② | **그림자 쐐기**(P-O3) — 빨강(회피 전용) |

> ⚠️ **밸런스 주의**: 엘리쟈는 노랑(패링) 패턴이 **원거리 1개뿐**이라 패링 그로기 충전 경로가 좁다. 단 §1.6상 광역(쿨 김)이 막히면 FAR/MID에서 원거리(암흑 구체)가 자주 나오므로 패링 기회는 확보된다. 그래도 느리면 P-O1 쿨 단축으로 조정 → MR-E.

| 광역 패턴 | **용암 분출**(P-Z3) — 3지점 동시 |

## 3. 패턴 상세 스펙

> **용어**: windup = 시전 속도(예고 게이지가 차는 시간) / resolve = 게이지 가득 참 = 판정 발생(= 근접 패링 유효 시점) / 쿨타임 = resolve 후 다음 패턴까지 후딜. 좌표·크기는 world unit(1 unit = 100px, 플레이어 = 가로 1 × 세로 2), 시간은 초.
> **히트박스 표기**: 근거리·광역은 **사각형(가로 W × 세로 H)** = 예고 게이지 영역과 동일(§1.5). 원거리는 투사체 반경·속도. **광역 쿨타임은 근접·원거리의 2~3배**(§1.6).

### 3.1 오르비스 — 엘리쟈 (원거리 1 + 광역 2)

| ID | 패턴 | 분류 | 색(패링) | windup | 쿨타임 | 히트박스 | 비고 |
|---|---|---|---|---|---|---|---|
| P-O1 | 암흑 구체 | 원거리 | 노랑(반사) | 0.6s | **1.0s** | 투사체 반경 0.2 · 속도 10 · **3발 동시(부채꼴)** | 보스→플레이어 방향 사각 게이지 후 3발 동시 발사. 유일한 패링 수단 |
| P-O2 | 암흑 강림 | 광역 | 빨강(불가) | 1.0s | **2.5s** | 사각 **가로 4 × 세로 3** | 플레이어 위치 추적 1곳. 빨강 사각 게이지(↑) |
| P-O3 | 그림자 쐐기 | 광역 | 빨강(불가) | 1.1s | **2.8s** | 보스 좌우 각 사각 **가로 4 × 세로 3** (2곳) | 보스 양옆 동시, **보스 발밑(중앙 가로 2) 안전** → 보스에게 붙도록 유도 |

- P-O1: 3발을 **동시에** 플레이어 방향 ±15° 부채꼴로 발사(순차 아님). 각 발 독립 투사체(반사 가능).
- P-O3: 좌우 협공으로 회피 공간을 좁히되 보스 발밑이 안전지대 → 플레이어가 근접하게 만들어(= CLOSE 밴드 유도) 다음 패턴과 연계.

### 3.2 엘나스 — 스노우맨 (묵직하고 느림)

| ID | 패턴 | 분류 | 색(패링) | windup | 쿨타임 | 히트박스 | 비고 |
|---|---|---|---|---|---|---|---|
| P-E1 | 눈주먹 강타 | 근접 | 노랑(가능) | 0.9s | **1.2s** | 전방 사각 **가로 3.6 × 세로 2** | windup 길어 패링·회피 여유, 대신 범위·발동 사거리 넓음 |
| P-E2 | 고드름 발사 | 원거리 | 노랑(반사) | 0.7s | **1.1s** | 투사체 반경 0.3(큼) · 속도 10 · 1발 | 큰 단발 투사체 |

### 3.3 폐광 — 자쿰 (변칙·다지점)

| ID | 패턴 | 분류 | 색(패링) | windup | 쿨타임 | 히트박스 | 비고 |
|---|---|---|---|---|---|---|---|
| P-Z1 | 팔 후려치기 | 근접 | 노랑(가능) | 0.7s | **1.0s** | 좌 또는 우 사각 **가로 4 × 세로 2** | 좌우 중 한쪽 팔만 판정 → 반대편 회피 가능. 긴 횡 사거리 |
| P-Z2 | 박수 레이저 | 원거리 | 노랑(반사) | 0.8s | **1.0s** | **수평 1발** 빠른 투사체(반경 0.2 · 속도 10, 빔 비주얼) | 박수 모션 후 플레이어 행(**수평 고정**)으로 1발. 반사 가능 |
| P-Z3 | 용암 분출 | 광역 | 빨강(불가) | 1.0s | **2.5s** | **3개 지점** 각 사각 **가로 3 × 세로 3** (동시) | 플레이어 위치 1곳 + 랜덤 2곳, 각 지점 빨강 사각 게이지(↑) |

#### P-Z2 박수 레이저 — 실제 메이플 패턴 각색 (조사 결과 반영)

- **원본(메이플 자쿰)**: 발판에 주황색 빛 예고(약 1.8초) → 자쿰이 **박수** → **수평 레이저**, 최대 HP 100% 고정 즉사, **2연타**, 박수 손 부분에도 판정, **양팔 건재 시에만** 사용([나무위키], [공략]).
- **본 게임 적용(각색)**: "레이저(빔)"는 본 게임의 **반사형 투사체 모델**과 직접 맞지 않으므로, **박수 후 플레이어 행(수평 고정) 방향으로 빠른 에너지 탄 1발**(기존 `Projectile` 재사용, `Parryable=true`)로 구현.
  - **방향**: **수평 고정**(플레이어의 좌/우 방향, y = 0). 플레이어가 다른 높이(발판)에 있으면 빗나감 → 높이 차로도 회피 가능.
  - **발수**: **1발**(원본 2연타는 단순화). 시각은 빔/레이저처럼 가늘고 긴 스프라이트, 판정·반사는 투사체.
  - **예고**: 박수 들어올림 모션 + §1.5 **원거리 게이지(보스→바깥)**.
  - **미도입(스코프 밖)**: 즉사 고정뎀(본 게임은 어차피 1피격 즉사라 무의미), 발판 행 한정, 팔 제거 기믹.

## 4. 구현 매핑 및 데이터 외부화

### 4.1 기존 코드 슬롯 매핑 (`BossController.mlua`, MR-B)

`BossController`의 더미 패턴 3슬롯(slam/tripleShot/redSmash)에 보스별 패턴을 리스킨하는 형태로 구현한다(공유 windup→resolve→delay 구조 유지).

| 코드 슬롯 | 기본 성격 | 엘리쟈 | 스노우맨 | 자쿰 |
|---|---|---|---|---|
| `tripleShot` | 원거리 투사체 | P-O1(3발 동시) | P-E2(1발) | P-Z2(박수 1발·수평) |
| `redSmash` | 광역(빨강·사각) | P-O2 | P-E3 | P-Z3 |
| `slam` | 근접(노랑·사각) | **P-O3로 리스킨**(광역②, `parryable=false`) | P-E1 | P-Z1 |

> - 엘리쟈는 근접이 없으므로 **`slam` 슬롯을 두 번째 광역(P-O3)으로 리스킨**(parryable=false, 좌우 2박스).
> - 발수(1/3발)·박스 크기·좌우/다지점·수평고정·parryable은 슬롯 파라미터로 분기. 색상은 `parryable` 플래그.
> - **근접 판정 원형 → 사각(박스)** 변경(`RegisterEnemyAttack` → `RegisterEnemyAttackBox` 계열, §1.5 예고=판정 일치).
> - **패턴 선택을 거리 밴드+우선순위+개별 쿨타임으로 교체**(현재 단일 phaseTimer+랜덤, §1.6).
> - **근접 패링 윈도우(§1.4)**: `TryParry` 근접 분기에 `hitAt ± ε` 시간 게이트(ε선행 0.15·후행 0.05).
> - **예고 게이지(§1.5) 신규**: windup 진입 시 생성, resolve 시 제거. 사망·그로기·그레이스 분기에서 잔류 정리(§4.4).

### 4.2 외부화 대상 (MR-F 연계)

보스별·패턴별 다음 값을 데이터/상수로 분리(하드코딩 금지):

- 보스: `MaxHP`, 그로기 최대치(= MaxHP/5), 추격 속도, **근접 발동 사거리**, 스프라이트/클립 RUID.
- 패턴: `windup`, `cooldown`(광역은 2~3배), `parryable`, 히트박스(사각 W×H / 투사체 반경·발수·속도·수평고정 여부), **광역 배치 사거리**(6), 예고 게이지 형태/색, SFX RUID.
- 전역: 패링 윈도우 ε(선행 0.15·후행 0.05, §1.4), 거리 밴드 경계(§1.6). (현재 `BossPatternDelay`·windup 일부 하드코딩 → 외부화 시 정리.)

### 4.3 서버 권위 (C-5)

그로기 게이지·100% 발동·3초 그로기·자동 감소·섬광탄 5초 감소 정지는 서버 권위, 클라 `@Sync` 동기화(`stage_design` §1.4.4). 패턴 판정(`RequestHit`)·패링 윈도우·거리 밴드 판정도 서버 측. 시간은 `ServerElapsedSeconds` 기준(슬로우 모션 무관). 예고 게이지 진행도 서버 windup 타이머에 종속, 클라는 표시만.

### 4.4 사망/재시작 정리

- 보스 사망(HP 0) 시 진행 중 투사체(P-O1 3발, P-Z2 등) **및 예고 게이지·잔존 광역 박스** 즉시 소멸(`stage_design` §1.4.4 / §2.5.3 단일 초기화 함수).
- ResolvePattern tail 자기파괴 가드 유지(MR-B에서 수정된 LEA-2011 회귀 방지): `if isvalid(self.Entity) == false then return end`.
- `CancelWindup`(그로기/그레이스)에서 예고 게이지도 함께 폐기.
