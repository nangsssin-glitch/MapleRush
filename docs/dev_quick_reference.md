## 2. 데미지 공식 (모두 ATK 기준 백분율)

| 공격 | 데미지 | 비고 |
|---|---|---|
| 평타 | `ATK × 100%` (=10) | 쿨타임 0.5초 |
| 대시 공격 | `ATK × 50%` (=5) | 일반 대시 중 |
| 강화 대시 공격 | `ATK × 150%` (=15) | 슬로우 모션 발동 대시, 대시 횟수 미소모 |
| 반사 투사체 | `ATK × 200%` (=20) | **명중 대상 무관 통일** (원발사자/타 적 동일) |
| 짱돌(아이템) | `ATK × 300%` (=30) | 적 투사체 사양 재사용 |
| 보스 그로기 충전(패링) | `(ATK × 100%) × 2` (=20) | 근거리·원거리 패링 공통, 공격력 강화 시 비례 증가 |

## 3. 플레이어 액션

### 3.1 평타 [P §2]
- 입력: **마우스 좌클릭** / 방향: 커서 방향 360°
- 판정: **플레이어 크기 2배 원형** (고정값), 근접 즉시 판정 (투사체 없음)
- 쿨타임: **0.5초** (쿨 중 입력 무시)

### 3.2 대시 [P §3.1]
- 입력: **우클릭 유지(조준) → 해제(실행)**, 방향 360°
- 이동 거리: **플레이어 길이 5배** / 이동 시간: **0.1초**
- **이동 시간(0.1초) 동안 무적(i-frame)** — 피격·즉사 면제, 강화 대시 포함, 이동 종료 즉시 해제 (실제 시간 기준)
- 대시 공격 데미지: `ATK × 50%`

### 3.3 슬로우 모션 [P §3.2]
- 발동 조건: 적 공격(또는 투사체)이 **플레이어 원형 감지 범위(플레이어 크기와 동일) 내 진입** + 그 순간 **대시** 실행
- 효과: **1초간(실제 시간)** 게임 시간 **0.25배속**
- 발동 시 대시는 **강화 대시**(`ATK × 150%`, 대시 횟수 미소모)
- **MSW 구현**: `_UtilLogic:SetClientTimeScale(0.25)` (클라이언트 전용; **기존 `_CameraService.TimeScale`은 오기 — 해당 속성 없음**). deltaTime만 변경, 룸 이동 시 자동 1.0 리셋. 스프라이트 자동 감속 미보장(C-2)
- **적·투사체 감속**: 서버 권위라 클라 timeScale만으로 안 느려짐 → **서버에서 `MovementComponent.InputSpeed`/투사체 속도/`SpriteRendererComponent.PlayRate`를 0.25배 직접 제어**(1인 1런 기준, V1 실측 확정). 코옵 도입 시 재설계 필요
- ⚠️ 플레이어 조작 요소(이동·점프·평타·대시·카메라·능력·UI·플레이어 사운드)는 **Wall-clock 독립 타이머**로 정상 배속 유지. 적·투사체·이펙트만 슬로우 적용.
- 시간원: `_UtilLogic.ElapsedSeconds`·`ServerElapsedSeconds` 모두 **timeScale 무관 실제 시간**(C-1 실측 완료). **느려지는 건 `OnUpdate`의 delta뿐(0.25배)** → 플레이어 타이머는 **ElapsedSeconds 스냅샷 차분**으로 구현, ⛔ delta 누산 금지

### 3.4 능력 [P §4]
- 슬롯: **1개** / 입력: **마우스 휠클릭**
- 사용 횟수: 일반 스테이지 **1회**, 보스 스테이지 **2회** (진입 시 초기화)
- 정의된 능력: 임시 무적(1초), 모든 투사체 무효화 *(목록 추후 확장)*

### 3.5 일회성 아이템 [P §7]
- 슬롯: **1~3번 숫자키** 고정 (자리 변경 미구현)
- 사용: **숫자키로 활성화 → 좌클릭으로 커서 방향 사용 → 같은 숫자키 재입력 시 비활성화**
- 한 번에 한 슬롯만 활성화
- ⚠️ **활성화 중 좌클릭 = 아이템 사용(평타 아님)**. 아이템 사용 후 평타는 평소대로 0.5초 쿨타임 적용(추가 패널티 없음)

| 아이템 | 효과 | 세부 |
|---|---|---|
| 짱돌 | `ATK × 300%` 투사체 | 커서 방향 발사, 적 투사체 사양·충돌/소멸 규칙 재사용 |
| 연막탄 | 은신 영역 5초 | **생성 위치 고정**(플레이어 3배 범위), 추격·조준 중이던 일반 적 즉시 해제, **보스는 은신 무시** |
| 섬광탄 | 화면 내 전 **일반 적** 3초 기절 | **보스는 기절 면역**; 대신 보스 그로기 게이지 **감소만 5초 정지**(충전 가능) |

## 4. 패링 시스템 [P §5]

- **자동 발동**: 평타 판정 범위와 적 공격(노란색) 히트박스가 겹치는 순간
- **노란색 = 패링 가능 / 빨간색 = 패링 불가** — **코드에선 적 공격/투사체마다 `parryable: boolean` 사전 지정**(true=노랑/패링·반사 가능, false=빨강/불가). 노랑/빨강 색상은 추후 지정할 공격 이펙트의 시각 표현이고, 판정 로직은 이 boolean으로 분기
- 대시 공격으로는 패링 **불가** (평타만 가능)

| 종류 | 효과 | 보스 그로기 충전 |
|---|---|---|
| 근거리 노란색 | 공격 무효화 (피격 없음) | 패링 즉시 `(ATK×100%)×2` |
| 원거리 노란색 | 투사체 반사 (반사 데미지 `ATK×200%`) | 반사체가 보스 **명중 시** `(ATK×100%)×2` |

- 반사 투사체: 플레이어 재피격 면제, 원래 투사체와 동일 히트박스/속도, 적 미명중 시 화면 밖 자연 소멸

## 5. 적 [S §1]

### 5.1 근거리 적 [S §1.2]
- 감지 범위: **플레이어 크기 3배** / 공격 범위: **1배**
- 행동: 감지 시 플레이어로 이동 → 공격 범위 내 진입 시 근접 공격
- **낭떠러지 낙하 없음** (땅 끝 정지; **MSW 기본 동작 아님 → `Raycast`/`Foothold.EndPoint`로 수동 발판 끝 감지 구현** — ✅ V2 실측 확정)

### 5.2 원거리 적 [S §1.3]
- 감지 범위: **플레이어 크기 5배** / 투사체 히트박스: **1/5배**
- 행동: 감지 시 **플레이어로 이동(접근)** + **조준 1초 → 발사 1개 → 쿨타임 1초**(사이클 약 2초) 반복
- 투사체: 속도 **10**, 직선 이동, **벽/플랫폼/플레이어 충돌 시까지** 진행 후 소멸
- 발사 조건: 플레이어가 감지 범위(5배) 내 (별도 근접 발사 거리 조건 없음)
- **낭떠러지 낙하 없음** (수동 구현 — §5.1과 동일)

### 5.3 보스 [S §1.4]
- 그로기 게이지 최대치 = **보스 전체(최대) 체력의 1/5**
- 충전: 일반 공격 = 명중 데미지량 / 패링 = `(ATK×100%)×2`
- 자동 감소: **초당 2%** (전투 중 일정, HP 무관) — **서버 권위, `ServerElapsedSeconds` 기준**(슬로우 영향 없음)
- 발동: 게이지 **100%** → 그로기 **3초** (이동·공격 정지)
- 패턴: **3~5개**(가변 풀, 구체 내용 미정) / 분노 페이즈·결정타 연출 **미도입**

## 6. 스테이지 구조 (거울 던전식) [S §2.2]

```
[테마팩 선택] → [노드 맵: 일반 N회(3분기 1택) → 상점(고정) → 보스(고정)] → [보스 클리어] → [다음 층 테마팩 선택] → 반복
```

- **테마팩 선택**: 각 층 시작 시. 미플레이 테마만 표시. 본 개발은 **2종 구현**(1층 2택 → 2층 1택)
- **일반 스테이지 분기**: 각 단계마다 3개 분기 중 1택 (현재는 전부 일반 스테이지로 임시 구현). 전진만 가능(되돌아가기 불가)
- **층별 일반 스테이지 횟수**: 1층 4회 / 2층 5회 / 공식 `4 + 1·(층수-1)`
- **최대 구현 목표**: 2층
- 노드 진입(모든 스테이지 진입) 시 플레이어 목숨 최대치 갱신

| 테마 | 구성 |
|---|---|
| 테마 1 (확정) | 오르비스 → 엘나스 → 폐광 |
| 테마 2 (미정) | 추후 결정 |

## 7. 상점 스테이지 [S §2.4]

| 품목 | 효과 | 초기 가격(메소) |
|---|---|---|
| 최대 목숨 추가 | **최대 목숨 +1** (회복 아님) | **100** |
| 스탯 강화 | 스탯 1종 강화 | **50** |
| 능력 획득/강화 | 보유 강화 또는 새 능력 | **150** |
| 일회성 아이템 | 짱돌·연막탄·섬광탄 등 | **40** |

- 구매: **좌클릭 → "정말로 구입하겠습니까?" 팝업 → 확정**
- 갱신: 구매 시 동일 종류가 **가격 2배**로 즉시 갱신 (무한, 메소 캡 내 오버플로우 방지)
- 종료: **나가기 버튼** → 다음 노드(보스)로 진행
- 최대 표시 품목: 5개 / 전투·사망 없음

### 8.1 일반 스테이지 (등장 확률, 독립 판정)
| 보상 | 확률 |
|---|---|
| 메소 | **100%** (클리어 시 자동 지급) |

- 전체 보상 목록이 **항상** 제시됨

- 일반: `일반 스테이지 클리어 수 × 10`
- 보스: `보스 스테이지 클리어 수 × 150`
- 적 처치 즉시 드롭 없음 → 클리어 시 일괄 지급

## 9. 메타 진행 [S §4]
- 게임 클리어 점수 = `일반 클리어 수 + (보스 클리어 수 × 3)`
- 게임 오버 시 점수 미지급 (클리어 시에만)
- 영구 강화는 게임 판과 독립 저장, 점수 캡 65,535
- 영구 강화로 변경된 기본값은 매 게임 시작 시 적용

## 10. 반드시 지킬 핵심 예외 규칙

> - **1피격 즉사**: 목숨은 HP가 아님. 피격 즉시 사망 → 목숨 1 감소 → 현재 스테이지 재시작
> - **목숨 갱신**: 스테이지 진입 시마다 최대치 복구. 게임 오버는 *한 스테이지 안에서* 목숨 소진 시에만 발생
> - **다중 피격**: 동시 다발 피격도 피격 판정 **1회만** 적용
> - **슬로우 모션 충돌 판정**: 플레이어/적 시간 스케일이 달라지므로 히트박스 겹침은 **실제 시간 기준 프레임**으로 처리
> - **아이템 효과 지속 시간은 실제 시간 기준**: 연막탄 5초·섬광탄 일반 적 기절 3초·섬광탄 보스 게이지 감소 정지 5초·짱돌 투사체 이동은 슬로우 모션 영향 없음
> - **일회성 아이템은 전투 스테이지에서만 사용 가능** (상점·보상 UI 중 사용 불가); 대시 이동 중(0.1초) 사용 불가
> - **클리어 판정**: 스폰 적 수 = 처치 적 수 카운터 일치 시 클리어 (잔존 적 0 보장)
> - **보스 사망 시 잔류 투사체·이펙트 즉시 소멸** (사후 피격 방지)
> - **반사 투사체는 플레이어 재피격 면제**
> - **대시 이동 중(0.1초)은 무적** — 피격/즉사 면제(강화 대시 포함), 무적은 이동 종료 즉시 해제
> - **적은 낭떠러지로 떨어지지 않음** (근거리·원거리 공통; **수동 구현** — `Raycast`/`Foothold.EndPoint`, MSW 기본 동작 아님)
> - **패링 오버랩 판정**: 평타 원형 ↔ 적 공격 히트박스/투사체 겹침은 자동 충돌이 아닌 **별도 오버랩 로직**으로 구현
> - **스테이지 재시작 = 단일 초기화 함수**: 적·투사체 소멸 + 슬로우 강제 해제 + 카운터/위치/목숨/임시메소 초기화를 한 함수에서 일괄 처리
